# 多阶段调色盘

## 前置条件

- `创作规划.yaml` 中 `characters[].personality.stages` 已记录各阶段创作方向
- `ejs.entries[].stages` 中每个阶段的 `condition` 已确定
- 基础调色盘概念：见 `personality-palette.md`
- 二次解释写法：见 `rephrase.md`
- 角色阶段原则：见 `composition.md`

## 什么时候用

`personality.stages` 不为空时走多阶段流程。此文档的输出替换单阶段 `personality-palette.md` 的产出。

## 核心约束

- 底色不随阶段变化
- 阶段专属衍生是该阶段**新出现**的行为，不是上一阶段衍生的升级版
- 通用衍生跨阶段始终存在；用户标记哪些是通用、哪些是阶段专属
- 二次解释也按通用 vs 阶段专属区分
- 引导用户自己创作衍生和二次解释，不通顺/重复的内容保留原样（详见 `personality-palette.md`、`rephrase.md`）

## 引导流程

### 第一步：确定底色

从 `创作规划.yaml` 读取第一个阶段的创作方向，引导用户写出底色。底色不随阶段变化，写一次即可。

### 第二步：初始阶段主色调/点缀/专属衍生

引导用户写出第一阶段的主色调、点缀和专属衍生。写法约束见 `personality-palette.md`。

### 第三步：标记通用衍生

阶段专属衍生写完后，询问用户哪些是**始终如此**的（通用），哪些是**只在这个阶段**的（专属）。

### 第四步：后续阶段

对每个后续阶段，依次：

1. 主色调/点缀是否有变化？逐项询问
2. 引导写出该阶段的新专属衍生
3. 同一行为在多阶段表现不同 → 归通用衍生，差异在二次解释中说明

### 第五步：通用二次解释

针对通用衍生中容易被 AI 误解的性格点，逐条写二次解释。写法见 `rephrase.md`。

用户不知道怎么写时，反问："你写了[某衍生]，AI 可能会怎么理解？你真正的意思是什么？"

### 第六步：阶段专属二次解释

对每个阶段的专属衍生，引导用户写阶段专属二次解释。

### 第七步：写总结

```
这就是[角色名]的性格调色盘，在这个调色盘上有着无数的颜色，任何时候都是由多种性格、行为、回忆组合驱动着[角色名]，并非单纯的一种颜色、一个标签。
```

## 自查清单

- [ ] 底色不随阶段变化，只写一次
- [ ] 每个阶段的主色调、性格点缀都有对应的衍生
- [ ] 阶段专属衍生是该阶段新出现的行为，不是上一阶段衍生的升级版
- [ ] 通用衍生跨阶段始终存在，放在所有条件外
- [ ] 阶段判定条件的边界值不重叠不遗漏
- [ ] 每个阶段的二次解释覆盖该阶段所有性格点
- [ ] 通用二次解释放在所有条件外，阶段专属二次解释用条件包裹
- [ ] 引导用户自己创作衍生和二次解释，不替用户写
- [ ] 通用自查：按 `references/rules.md` 通用自查
