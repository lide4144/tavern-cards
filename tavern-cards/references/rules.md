# 写作规则

所有世界书条目编写必须遵守的规则。本文档是唯一的写作规则来源，其他文档不再重复。

## 格式规范

- 使用 YAML 中文格式
- 缩进用 2 空格
- 用 xml 标签包裹条目内容（如需要）
- 数据库格式优先，不是散文：用列表和键值对，不用段落
- 语法冲突规避 / 多条目合并：参见 `references/conventions.md`
- **全文简体中文**：所有条目内容、变量名、文件名使用简体中文，不使用繁体字或日文汉字（如"桜"→"樱"、"宮"→"宫"、"靜"→"静"、"繪"→"绘"）。转化阶段需检查源材料，在转为条目内容前统一替换为简体
- **禁止占位符**：条目内容中不出现"某城市""某学校""某组织"等含糊表述。所有关键具体名称（地名、校名、店名、角色名等）必须明确写入。不确定时暂停创作并询问用户，不得自行捏造或用模糊词替代

## 绝对零度原则

- 有主观评价和判断 → 删除
- 有陈旧/劣质比喻 → 改成直接描述
- 堆砌无意义形容词 → 精简（"红色的门"可以保留，"精美绝伦的门"需要精简）
- 用代词和意象词导致含义模糊 → 改成具体本意

## 八股化检查

发现以下表达时删除或改写：

- 模糊词：似乎、几乎、仿佛、如同、宛如、好像
- 陈旧比喻：像小兽、像小兔子、投石入湖、心湖泛起涟漪
- 八股微表情：嘴角微微上扬、眼中闪过一丝 XX
- 语气声线描写：带着 xx 的口吻、用 xx 的语气
- 极端情绪词：陷入极大的恐惧、极度羞耻、万念俱灰
- 否定转折句式：不是...而是...的固定句式
- 过度心理描写：大段内心活动
- 性格标签化："她很温柔""她很善良"

注意：正常使用"笑了""皱眉""攥拳"等常见动作词不算八股，不要矫正过度。白描是简洁直接，不是物理学描述。

## 具体性检查

- 有抽象描述 → 改成具体行为
- 有笼统表述 → 改成具体细节
- 贴标签 → 改成行为展现

错误："性格温柔善良" → 正确："遇到受伤的小动物会带回家照顾，会把零花钱分给流浪汉"

## 通用写作要点

- **外貌只写特征**：只写偏离 AI 数据库默认认知的部分。判断标准：遮住名字能否认出是谁
- **背景只写关键事件**：删掉这条背景，角色会不会变？不会 → 删
- **关系写具体画面**：不写"深厚的感情"，写"有记忆起就在一起"

## 世界观压缩

- 连接词用冒号/逗号替代
- 不写主观评价
- 不写 AI 已知信息
- 只写让 AI 会出错的差异信息

对世界观条目的每句话问：
1. 删了这句 AI 会错吗？不会 → 删
2. 是信息还是装饰？装饰 → 改写或删
3. 列表能替代吗？能 → 改列表
4. 不看原文能理解吗？不能 → 补关键信息

## 通用自查

- [ ] 无主观评价和判断（绝对零度）
- [ ] 无模糊词/陈旧比喻/八股微表情（八股化）
- [ ] 无抽象描述/笼统表述/标签（具体性）
- [ ] 世界观条目通过压缩检查
- [ ] 全文简体中文：无繁体字、日文汉字
- [ ] 无占位符或者含糊表述
