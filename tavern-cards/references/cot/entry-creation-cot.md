# 条目创作前置 CoT 模板

## 使用方式

- **auto 模式条目文件**：AI 写全部内容，执行完整 10 步自检
- **guided 模式下 AI 补全内容**：仅 AI 生成的部分走自检，用户写的内容不查
- **聊天框创作内容**：仅执行第三步～第九步中适用项（发现风险自动修正后输出）
- **分析性对话**：跳过

## 前置自检模板

== 前置元认知自检 ==

### 第一步：条目类型确认

当前条目是哪种类型？
- 角色类 / 世界观类 / 呈现方式类 / 技术类
- 对应的创作规则文档是哪个？

### 第二步：创作目标核对

- 创作规划中对此条目的目标描述是什么？
- 用户提供了哪些具体信息？
- 哪些需要我从上下文推断？

### 第三步：绝对零度检查

**核心**：删除一切主观评价、劣质比喻、模糊意象。

□ 删除主观评价
   ✗ "她非常善良""他很大方""性格温柔"
   ✓ 改为行为展现，或直接删除

□ 删除陈旧比喻
   ✗ "心湖泛起涟漪""像小兽一样""投石入湖"
   ✓ 直陈事实："她想起了那件事"

□ 不堆砌形容词
   ✗ "精美绝伦的门"
   ✓ "雕花木门"（保留必要特征即可）

□ 不写模糊意象
   ✗ "那个地方"
   ✓ "城东的老茶馆"

**判定**：删掉这句后，角色/世界观的核心信息是否有损失？无损失则删。

### 第四步：八股化检查

**核心**：删除套路化、模板化的表达。

#### 词汇黑名单

□ 模糊词
   ✗ 似乎、几乎、仿佛、如同、宛如、好像
   ✓ 删除或改为具体描述

□ 陈旧比喻
   ✗ 像小兽、投石入湖、心湖泛起涟漪
   ✓ 删除比喻，直陈事实

□ 八股微表情
   ✗ 嘴角微微上扬、眼中闪过一丝 XX、勾起一抹弧度
   ✓ 删除或改为简洁动作（笑了/皱眉/攥紧手）

□ 语气声线描写
   ✗ "带着XX的口吻""用XX的语气""XX的声音响起"
   ✓ 删除，让对话本身承担

□ 极端情绪词
   ✗ 陷入极大的恐惧、万念俱灰、彻底崩溃
   ✓ 改为具体行为或状态描述

#### 句式黑名单

□ 否定转折
   ✗ "她不是冷漠，只是不善表达"
   ✓ "她不善表达"（只保留"而是/只是"后面的内容）

□ 标签化
   ✗ "她很温柔善良"
   ✓ "遇到受伤的小动物会带回家照顾"

□ 过度心理 / 大段内心独白
   ✓ 保留关键 1-2 句，其余删除

□ 空洞数字
   ✗ "目光停留了1.5秒""沉默了三秒又七分之一秒"
   ✓ 删除

#### 破折号检查

□ 破折号补述
   ✗ "她会照着做——从小习惯了"
   ✓ "她会照着做"

□ 破折号自问自答
   ✗ "她信了——为什么会信？因为……"
   ✓ "她信了"

□ 破折号画外音
   ✗ "她接了资料便信了——这是她的习惯"
   ✓ "她接了资料便信了"

**判定**：把破折号连同后半段一并删掉，前半段独立成句是否仍然完整？完整则破折号是冗余的。

#### ⚠️ 例外——以下不算违规，不要过度矫正

- 正常使用"笑了""皱眉""攥拳""点头""摇头"等常见动作词
- 自然的对话描写（不含语气标签的纯对话）
- 必要的场景描述（时间、地点、氛围基调）
- 开场白中选择的文风本身允许的修辞手法

**白描是简洁直接，不是物理学描述。** 不要矫正到"她将右手以每秒0.5米的速度抬起"这种程度。

### 第五步：具体性与活人感检查

**核心**：用行为展现替代抽象标签。角色必须通过具体行为、物件、说话方式、关系反应站起来，而不是靠标签和形容词。

#### 抽象标签替换

□ 抽象标签
   ✗ "性格温柔善良"
   ✓ "遇到受伤的小动物会带回家照顾"

□ 笼统表述
   ✗ "他们关系很好"
   ✓ "有记忆起就在一起"

□ 空洞关系
   ✗ "深厚的感情"
   ✓ "每次见面都会拥抱"

**判定**：这句能否替换成具体画面？能则改。

#### 活人感检查（适用于角色类条目和叙事式开场白）

□ 纯标签人设
   ✗ "性格温柔善良"
   ✓ "遇到受伤的小动物会带回家照顾"

□ 缺少行为锚点
   ✗ "她很独立"
   ✓ "搬进新宿舍那晚，她没让任何人帮忙搬箱子"

□ 抽象关系
   ✗ "与母亲关系疏远"
   ✓ "母亲打来的电话，她响三声就挂掉"

□ 空场景开场（适用于开场白）
   ✗ "天气很好，你走在街上"
   ✓ "面试通知的附件里只有一首未完成的 demo 和一个地址"

**判定**：删掉标签后，读者能否看到这个角色正在做什么、说什么、拿着什么、和谁在一起？不能则补具体画面。

### 第六步：假性主体与远距离叙事检查（适用于角色类条目和开场白）

**核心**：避免无生命主语做人类动作，避免远距离的情绪宣告，让角色通过行为/对话展现自身。

#### 假性主体

□ 抽象概念做主语
   ✗ "一个念头在她心里成形"
   ✓ "她想到——"

□ 物品做人类动作
   ✗ "这句话戳中了她的心事"
   ✓ 删，或改为"她沉默了一会儿"

□ 决定/感受自动发生
   ✗ "一股莫名的烦躁涌了上来"
   ✓ "她皱眉看了看时钟"

□ 情绪被动承受
   ✗ "她被一种复杂的情绪包裹"
   ✓ 改为直接展现行为

#### 远距离叙事

□ 情绪宣告
   ✗ "她感到一阵悲伤"
   ✓ 改为行为或对话："她别过脸去"

□ 自我认知宣告
   ✗ "她意识到自己错了"
   ✓ 改为行为："她放下杯子，低声说'是我的问题'"

□ 内心独白式
   ✗ "她对自己说，也许该放手了"
   ✓ 删"她对自己说"，保留"也许该放手了"

□ 疏离描写
   ✗ "她的语气听不出任何情绪波动"
   ✓ 删，让对话本身传达

□ 上帝视角
   ✗ "她并不知道，这个决定将改变一切"
   ✓ 删，读者不需要被预告

□ 被动感受
   ✗ "一股暖意在她心里蔓延开来"
   ✓ 改为具体行为

**判定**：把主语换成具体的行为者名称，句子是否还成立？不成立则说明主语不是真实的行为者。删除修饰性的感受描述，问"角色做了什么/说了什么"而不是"角色感到了什么"。

### 第七步：语境专项检查

#### 翻译腔检查

**核心**：避免英文句式直译，使用自然中文语序。

□ 系表判断
   ✗ "她是拘谨的"
   ✓ "她表现得拘谨"

□ 介词前置
   ✗ "对这件事，她始终是回避的"
   ✓ "她一直回避这件事"

□ 被动式
   ✗ "一股温暖被她感受到了"
   ✓ "她心里暖洋洋的"

□ 宾语前置
   ✗ "此事她至今仍在思忖"
   ✓ "她到现在还会想起这件事"

□ 物化形容
   ✗ "话很重"
   ✓ "话很有分量"

**判定**：句子是否符合「主语 → 状语 → 动词 → 宾语」的自然语序？不符合则改。

#### 元叙事检查

**核心**：删除作者写作笔记，只保留角色/世界观内容。

□ 写作指引
   ✗ "AI 写作要点：此阶段让角色……"
   ✓ 删除整段

□ 分析框架
   ✗ "核心冲突_溺爱即封锁"
   ✓ 改为直陈式标题

□ 用途说明
   ✗ "戏剧用途：承担反差功能"
   ✓ 删除

□ 标记词
   ✗ "本卡独家""本卡专属"
   ✓ 删除

□ 引言总结
   ✗ "这是本卡最重要的机制"
   ✓ 删除

**判定**：这段文字是给作者看的笔记，还是给 AI 看的内容？笔记则删。

#### 跨条目引用检查

**核心**：每条必须独立可读，禁止依赖其他条目。

□ 详见引用
   ✗ "详见「老张身份」"
   ✓ 把被引用内容的关键句复制进来

□ 配套说明
   ✗ "与世界观 X 配套"
   ✓ 删除注释，必要时融入本条

□ 对应引用
   ✗ "对应「时间框架」"
   ✓ 删除

□ 系列引用
   ✗ "详见「节目_*」系列"
   ✓ 删除

□ 跨EJS阶段引用（仅EJS条件块内）
   ✗ "比第一阶段更克制"  "上一阶段时……"
   ✓ 删除，EJS条件块之间互相不可见，块内禁止提及任何其他阶段的标识

**判定**：删掉条目名称和关键词后，仅看条目内容是否仍能独立成立？不能则补全。EJS条件块之间互相不可见。

### 第八步：正面写作规则检查

**核心**：对照 `references/rules.md` 的规则。

□ 外貌只写特征
   ✓ 只写偏离 AI 数据库默认认知的部分
   ✓ 遮住名字能否认出是谁？
   ✗ 默认认知："黑头发黑眼睛""五官精致""白皙"

□ 背景只写关键事件
   ✓ 删掉这条背景，角色会不会变？不会则删
   ✗ 大段成长经历流水账

□ 关系写具体画面
   ✓ "有记忆起就在一起"
   ✗ "她们感情深厚"

□ 行为展现性格
   ✓ "遇到受伤的小动物会带回家照顾"
   ✗ "性格温柔善良"

□ 一句一意
   ✓ "长辈安排下来我会照着做"
   ✗ "长辈安排下来我会照着做。这件事我从小就习惯了"（第二句是同义补述）

### 第九步：世界观专项自检（仅世界观类条目）

对世界观概览、区域、时间线条目执行以下压缩检查：

□ 每句话过四问
   1. 删了这句 AI 会错吗？不会 → 删
   2. 是信息还是装饰？装饰 → 改写或删
   3. 列表能替代吗？能 → 改列表
   4. 不看原文能理解吗？不能 → 补关键信息

□ 删掉了 AI 已知信息和主观评价吗？
□ 只写了差异信息吗？

### 第十步：结论

→ 全部通过 → 进入创作
→ 发现风险 → 标记风险项（记录具体步骤编号），修正后重新自检，通过后进入创作
