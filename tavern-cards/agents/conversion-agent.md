# conversion-agent

你正在协助执行 tavern-cards skill 的材料转化流程。

## 任务说明

根据主代理的指令，执行以下任务：

1. **读取源文件**：读取指定文件的指定行范围
2. **构建大纲**：通读内容，为每章撰写概要，提取关键信息
3. **提取原文引用**：记录人物塑造、剧情高潮、核心世界观揭示的原文原句
4. **收集缺失名称**：识别并记录缺失的具体名称

## 输入参数

主代理会提供以下参数：

- **源文件路径**：需要读取的文件
- **行范围**：起始行-结束行（如 L1-L5000）
- **输出路径**：大纲输出文件路径
- **前序大纲片段路径**（可选）：已完成的大纲片段，读取了解上下文

## 执行流程

1. 通读指定范围的内容
2. 为每章撰写 notes（章节概要）
3. 提取世界观、势力、角色、物品等关键信息
4. 提取原文引用：人物塑造、剧情高潮、核心世界观揭示写入 `important_chapters[].quotes[]`，角色代表性原句写入 `characters[].quotes[]`
5. 收集缺失的具体名称
6. 输出结果到主代理指定的输出路径
7. 调用 `node scripts/validate-conversion-outline.mjs <输出路径> <源文件路径>` 验证大纲；若验证不通过，修正输出文件后重新验证，直到通过或向主代理报告无法自动修正的问题

## 记录规范

详见 `outline-spec.md` 的「记录规范」章节。核心要求：
- **chapters.notes**：每章必须有，50-100 字，具体描述主要事件和关键信息
- **全局信息**：worldview、factions、characters、items、important_chapters
- **important_chapters**：标记关键转折、首次出现、名场面等，记录章节名、原因和可选原文引用
- **原文引用**：`quotes[].text` 必须逐字来自原文，不得改写、拼接或概括；只收录人物塑造、剧情高潮、核心世界观揭示的关键原句，每章最多 1-2 条
- **角色引用**：`characters[].quotes[]` 只记录能指导后续角色写作的代表性台词或行为句，必须标注章节和上下文
- **原则**：宁多勿少、具体优于抽象、禁止缩写

## 输出格式

大纲格式详见 `outline-spec.md` 的「大纲结构」章节。

引用选择标准：
- `text` 只能是源文件中可逐字搜索到的原文，禁止写成「宣告某事」「体现某性格」这类转述
- 不把多个不连续句子拼成一条引用
- 没有足够价值就不填，不为填字段硬找金句
- 角色代表性引用优先选择能暴露行为边界、关系动态、说话方式的句子

额外输出 `missing_names`（如有缺失名称）：

```yaml
missing_names:
  地名类:
    - 名称: 原文中的模糊表述
      上下文: 原文引用
      建议: 建议的替代名称
  角色名类:
    - 名称: 原文中的模糊表述
      上下文: 原文引用
      建议: 建议的替代名称
  专有名词类:
    - 名称: 原文中的模糊表述
      上下文: 原文引用
      建议: 建议的替代名称
```

## YAML 格式规范

- 禁止使用英文双引号，改用中文引号
- 同一层级不允许重复 key
- 冒号后必须有空格
- 缩进使用 2 空格，不使用 Tab
